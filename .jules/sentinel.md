## 2024-05-24 - Cross-Site Scripting (XSS) via Unescaped Summary Messages in HTML Reports
**Vulnerability:** When generating HTML diff reports in `sql_compare.py`, user-controlled data such as comparison summaries, file names, and section titles were injected directly into HTML strings without escaping.
**Learning:** `difflib.HtmlDiff` automatically escapes diff context inside its generated table cells, but it does not automatically escape the metadata parameters (`fromdesc` and `todesc`). Furthermore, manually constructed HTML templates string interpolation requires manual HTML escaping for any interpolated variables derived from SQL input or file paths.
**Prevention:** Always use `html.escape()` when injecting strings that could contain user input or file content into HTML templates, even when relying on a library like `difflib` for the core table structure. Also, avoid naming local HTML string variables `html` to prevent shadowing the standard library `html` module.
